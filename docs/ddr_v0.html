Differential Drive Robot (DDR)

Introduction

Costruire un sistema software che comanda un Differential Drive Robot (DDR).

Requirements

Costruire un sistema software che comanda un Differential Drive Robot (DDR) in modo che, partendo dalla posizione iniziale HOME, il robot si sposti lungo il perimetro di una stanza rettangolare vuota.
Il DDR si deve fermare quando tornato in HOME.

Il DDR è anche sensibile ai dati rilevati da un Radar.
Quando il Radar rileva un 'intruso' a una distanza minore di una distanza prefissata ``DMIN``:
  • se il DDR si sta muovendo, il DDR si ferma fino a che il Radar non rileva più un 'intruso' così vicino
  • se il DDR è fermo in HOME, il DDR ruota su sè stesso fino a che un 'intruso' così vicino scompare

Requirement analysis

Il sistema software è composto dalle seguenti entità e proprietà:

  • la Stanza e HOME
    Per formalizzare la stanza, la modello come un rettangolo in un piano euclideo di dimensioni B (base) e H (altezza). Per rendere il modello indipendente dalle unità di misura fisiche e più vicino alla logica di movimento, introduco l'unità di misura "unità robotica" (R).
    • Stanza: formalizzata tramite le variabili B (larghezza), H (altezza) e R (lato del robot).
    • HOME: rappresenta la cella di partenza e arrivo del percorso. Viene modellata come una variabile di stato (coppia di coordinate (x, y)) all'interno di una griglia che rappresenta la stanza.
    Si formalizza la struttura della stanza con il seguente codice Kotlin: 
    data class Stanza(val B: Int, val H: Int, val R: Double) {
val home: Posizione = Posizione(0, 0)
}

data class Posizione(val x: Int, val y: Int)
  • Il Radar
    Il Radar è un'entità di input che rileva un intruso a una distanza D. Dal punto di vista della modellazione, il Radar deve rendere disponibile questa informazione al sistema.
    Esistono due modi per formalizzare questa interazione (modello dell'analista):
    1. modello a eventi: il Radar emette periodicamente un segnale asincrono. 
      Event radar : distance(D) // D in metri
    2. modello a richiesta: il sistema interroga il Radar quando necessario. 
      Request getDistance : getDistance(D)
Reply distance : distance(D) for getDistance
  • DDR Robot e Mind
    Dall'analisi emerge una distinzione fondamentale tra il corpo e la mente del sistema:
    • DDR Robot: è l'attuatore fisico. In questa fase lo considero un'entità puramente reattiva che risponde a comandi (avanti, ruota, stop). Non possiede logica decisionale propria.
    • Mind (Business Logic): è l'attore proattivo che contiene la logica per percorrere il perimetro e gestire le interruzioni del Radar.
Sulla base di quanto analizzato sopra, si formalizza la struttura del sistema con il seguente modello QAK iniziale: 
System ddr_perimeter_system

Event radar : distance(D) // modello a eventi per il radar
Dispatch move : move(CMD) // CMD = forward, backward, rotate, stop

Context ctxddr ip [ host="localhost" port=8040 ]

// Il Radar emette dati sulla distanza
QActor radar context ctxddr {
    State s0 initial { }
}

// Il DDR agisce come attuatore 
QActor ddr context ctxddr {
    State s0 initial { }
}

// La "Mente" contiene la logica del perimetro e la gestione DMIN
QActor mind context ctxddr {
    [# var DMIN = 0.5 #]
    State s0 initial { }
}
